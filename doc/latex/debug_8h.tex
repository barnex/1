\subsubsection[{debug\_\-verbosity}]{\setlength{\rightskip}{0pt plus 5cm}void debug\_\-verbosity (int {\em level})}}
\label{debug_8h_a624a10b4d886cb41e9bfd15f3ab1b1e8}
Setting the verbosity controls how many debug messages are displayed.


\begin{DoxyItemize}
\item The default, 1, means only \hyperlink{debug_8h_ab0b439a41f42a310f0e37b337db7b7fe}{debug()} messages are shown.
\item Level 2 is more verbose: also \hyperlink{debug_8h_a958bd3181af67ca831697af3936abadc}{debugv()} messages are shown.
\item Level 3 means full verbosity: even \hyperlink{debug_8h_a97079126c32d88d3f950c6acd183fd11}{debugvv()} messages are shown, which may cause a huge amount of output.
\item Level 0 means be silent, show no messages at all.
\end{DoxyItemize}

Example: 
\begin{DoxyCode}
    debugv( fprintf(stderr, "size = %d \n", size) );
    beenhere();     // Prints: "Been here: file XXX.c, line YYY (only for verbosi
      ty 3)
\end{DoxyCode}


\begin{DoxyNote}{Note}
Do not worry about performance. When compiling with NDEBUG defined, all debug messages are suppressed and the compiler should throw away the calls to empty functions. (todo) 
\end{DoxyNote}
